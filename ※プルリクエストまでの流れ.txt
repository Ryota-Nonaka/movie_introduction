
■初回■

# git clone <URL>　(※既にローカルにフォルダがある場合　→　# git pull )

# git branch "ブランチ名"		//今回の変更用のブランチを作成

# git checkout "ブランチ名"		//作成したブランチに移動　はじめは"master"に設定されていると思う

# ※ソースコードを変更

# git add --all				//コミットしたいファイルを全てコミットの対象にする

# git commit -m "コメント"		//変更履歴を生成

# git push -u origin "ブランチ名"	//ローカルリポジトリの内容をリモートリポジトリに反映する

# ※コード変更をした人が、github画面でプルリクエストを作成する
　	(※管理者にメールが届きます)

# ※管理者が、プルリクエストを承認して、変更内容を"master"にマージする
　	(※依頼者にメールが届きます)

# git pull				//リモートリポジトリの最新の状態をローカルへ更新

# git checkout master			//ローカルのブランチを"master"へ切り替え

# git branch -d "ブランチ名"		//マージ済みのローカルのブランチを削除


■開発途中にマージする場合■

# ※ソースコードを変更

# git add --all　　			//コミットしたいファイルを全てコミットの対象にする

# git commit -m "コメント"		//変更履歴を生成

# git push "ブランチ名"　		//ローカルリポジトリの内容をリモートリポジトリに反映する

# ※管理者にソースコードのチェックをしてもらう



(その他コマンド)
